Projets Front

OnTheSpot

Le projet OnTheSpot associe 2 technologies React et Symfony. Le projet a vu le jour avant la dernière partie de la formation: l’apothéose. C’est l’immersion dans le monde du travail avec ses collègues de promotion afin de réaliser un projet à présenter sur Youtube. Nous avons travaillé sur ce projet à 5 (3 en front et 2 en back) et je tenais le rôle de lead dev front au sein de l'équipe. Pour pimenter un peu le challenge, la durée de l’apothéose est de 1 mois. Yann Demor est à l'initiative de ce projet, améliorer l’expérience de la plage, avec des petites douceurs. L’idée originale est de créer une application e-commerce qui permettrait de se faire livrer snacks, boissons ou autres produits directement sur la plage. L’utilisateur n’aurait qu'à passer commande sur son téléphone et quelques instants plus tard, il pourrait les savourer tout en profitant du soleil en n’ayant bougé que le petit doigt. Étant trop compliqué à réaliser dans le délai imparti, il a fallu revoir le projet, les priorités. OnTheSpot devient un e-commerce avec point de retrait. On garde l’aspect pratique, qui permet à l’utilisateur de ne pas faire la queue. Le back-office permet au vendeur de modifier les produits/commandes et ainsi de gérer son magasin.

Projets Back

OnTheSpot-Nest

Le projet OnTheSpot-Nest est de le reflet de mon apprentissage du Framework Nest.js. Nest.js est Framework basé sur Express permettant de créer des applications Node.js hautement scalables, testables et facilement maintenable. N'ayant pas d'éclair génie pour créer une nouvelle API innovante, j'ai décidé de reprendre l'API de OnTheSpot faite sous Symfony pour m'exercer. Ce projet est en cours de développement et pour des raisons de sécurité l'ouverture à la création de compte n'est pas encore ouverte. Vous pouvez tout de même utiliser les endpoints non protégés pour récupérer les données.

Réalisation

OnTheSpot-Nest a été réalisé avec Nest.js en TypeScript, avec les outils suivants:
- NestAdmin.js: pour le back-office (librairie non maintenue future migration vers Admin Bro)
- TypeORM: ORM typé utilisé pour la communication avec la base de donnée.
- Swagger: librairie permettant de produire une documentation sur les différents endpoinds et des tests associés. (permet une visibilité immédiate de l'API ainsi que la substition d'outils comme Insomnia ou Postman)
- Passport: C'est un middleware d'authentification pour Node.js. Utilisé pour l'authentification avec des Tokens Jwt.
